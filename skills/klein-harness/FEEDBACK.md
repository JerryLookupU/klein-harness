# Feedback

如果你试用了这套仓库，最有价值的反馈不是“好不好”，而是“哪里卡住了”。

## 最希望收到的反馈

1. 你试用的项目是什么类型
2. 你用了哪些部分
3. 你在哪一步卡住
4. 你最想删掉哪一段
5. 你最想保留哪一段
6. 你觉得哪个对象命名最难懂
7. 你觉得哪个脚本最该变成默认入口

## 建议反馈格式

```md
### 背景
- 项目类型：
- 使用模型：
- 是否并发：

### 使用到的内容
- 文档：
- 脚本：
- 状态文件：

### 卡点
- 卡在哪一步：
- 预期是什么：
- 实际发生了什么：

### 建议
- 可以删掉的内容：
- 应该更早出现的内容：
- 应该自动化的内容：
```

## 建议附带的证据

- 相关命令
- 相关 `.harness/` 文件片段
- query/dashboard 输出
- 哪个 prompt 让模型跑偏
- 哪个 session / task / worktree 让你困惑

## 最重要

如果你只能提一条反馈，请优先提：

**哪一个地方最影响你真正把它用起来。**
