# Testing Policy

- 测试默认使用真实数据、真实 API key、真实本地环境进行验证，并尽量按照上线环境的路径、依赖、配置和行为执行。
- 不要优先引入 fake 环境、mock 数据、print mode、伪造 session 或额外测试实体来替代真实链路，除非用户明确要求，或真实环境因外部条件不可用而被明确标记为 `BLOCKED`。
- 当仓库已经接近上线态时，优先直接验证实际结果，避免为了测试再增加一层虚拟适配、镜像实体或额外抽象。
- 如果某项测试只能通过 fake/mock 完成，必须先说明原因、记录与线上环境的差异、列出后续需要移除或对齐的项，再继续执行。
- 测试结论必须明确区分：
  - 真实环境验证通过
  - 仅 fake/mock 验证通过
  - 因真实环境不可用而阻塞
